# A股量化交易工具产品需求文档

## 1. 产品概述

### 1.1 产品愿景

打造一款面向个人量化投资者和小型团队的专业级A股量化交易平台。该平台集策略研发、回测分析、实盘交易与风险管理于一体，旨在提供高性能、高可靠性、用户友好的量化投资解决方案。

### 1.2 目标用户

| 用户群体 | 特征与需求 |
| :--- | :--- |
| **个人量化爱好者** | 具备一定的编程和金融知识，寻求功能强大、易于上手的工具来验证和执行自己的交易想法。 |
| **小型私募/投资团队** | 需要稳定、高效、可扩展的系统来管理多个策略和账户，对交易执行速度和风控有较高要求。 |
| **金融/计算机专业学生** | 希望通过一个现代化的平台学习和实践量化交易知识，需要丰富的学习资源和活跃的社区。 |

## 2. 功能规划 (MVP - 最小可行产品)

我们将采用敏捷开发模式，分阶段实现产品功能。第一阶段（MVP）将聚焦于核心功能的闭环，确保用户可以完成从策略开发到模拟交易的全过程。

### 2.1 核心功能模块

| 模块 | 功能描述 | 优先级 |
| :--- | :--- | :--- |
| **数据服务 (Data Service)** | 提供稳定、准确的A股历史日线、分钟线行情数据及基础财务数据。 | **高** |
| **策略开发 (Strategy IDE)** | 内置基于Web的 Monaco Editor，支持Python语言，提供语法高亮、代码补全和策略模板。 | **高** |
| **回测引擎 (Backtesting Engine)** | 支持基于历史数据的向量化回测和事件驱动回测，输出标准化的回测报告。 | **高** |
| **模拟交易 (Paper Trading)** | 对接实时行情，模拟真实市场环境，支持模拟账户的交易和盈亏计算。 | **高** |
| **用户界面 (Dashboard UI)** | 参考TradingView，提供行情图表、观察列表、账户信息、回测历史等核心可视化组件。 | **高** |

### 2.2 详细功能点

#### 2.2.1 数据服务

- **数据源**：初期优先整合免费、稳定的数据源，如 `AkShare` 或 `Tushare` 的免费接口。
- **数据覆盖**：覆盖全A股市场，提供至少过去10年的日线数据和过去2年的分钟线数据。
- **数据更新**：日线数据盘后更新，分钟线数据在模拟盘中实时更新。
- **财务数据**：提供核心财务报表数据（利润表、资产负债表、现金流量表）。

#### 2.2.2 策略开发 (IDE)

- **编程语言**：支持 **Python 3.8+**。
- **核心API**：提供简洁的API，用于数据获取、下单、账户信息查询等。
- **策略模板**：提供海龟交易法则、双均线等经典策略模板，方便用户快速上手。
- **依赖管理**：支持用户在策略中引入常用的第三方库（如 `Numpy`, `Pandas`, `TA-Lib`）。

#### 2.2.3 回测引擎

- **回测模式**：支持基于Pandas的向量化回测，速度快，适合初步筛选；支持事件驱动回测，逻辑更精细，接近实盘。
- **回测报告**：生成包含以下关键指标的回测报告：
    - **收益指标**: 累计收益率、年化收益率
    - **风险指标**: 最大回撤、夏普比率、索提诺比率
    - **交易统计**: 总交易次数、胜率、盈亏比
- **可视化**：绘制收益曲线图，并与基准（如沪深300）进行对比。

#### 2.2.4 模拟交易

- **行情对接**：通过WebSocket连接实时数据源，获取实时行情。
- **撮合机制**：模拟交易所的撮合机制，处理限价单和市价单。
- **账户模拟**：模拟真实的资金、持仓、盈亏、手续费和滑点。

## 3. 非功能性需求

| 需求类型 | 具体要求 |
| :--- | :--- |
| **性能** | 回测引擎在处理10年日线数据时，单次回测应在30秒内完成。实时行情延迟应低于500ms。 |
| **可靠性** | 后端服务可用性需达到99.9%。所有交易和账户数据必须持久化存储，防止数据丢失。 |
| **安全性** | 用户策略代码和个人信息需加密存储。前后端通信采用HTTPS。 |
| **可扩展性** | 系统架构应采用模块化设计，便于未来增加新的数据源、交易品种（如期货、期权）和功能模块。 |
| **用户体验** | 界面响应迅速，交互逻辑清晰，提供暗色/亮色两种主题模式。 |
