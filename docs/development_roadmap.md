# QuantRust: 开发路线图 (Roadmap)

**版本**: 1.0
**作者**: Manus AI
**日期**: 2026-02-26

---

## 1. 概述

本文档为 QuantRust 项目的开发提供了分阶段的路线图。我们将采用敏捷开发的思想，通过多个 Sprint 迭代，逐步交付功能，并根据用户反馈进行调整。

## 2. 开发阶段划分

### **第一阶段: MVP 核心功能闭环 (预计 8 周)**

**目标**: 跑通从策略开发、回测到模拟交易的核心流程，交付一个可用的最小可行产品。

- **Sprint 1-2: 基础设施与核心架构 (2周)**
    - [ ] **后端**: 搭建 Rust 微服务框架 (Cargo Workspace)，配置 Docker 环境。
    - [ ] **后端**: 实现 `quantrust-core` 模块，定义核心数据结构。
    - [ ] **后端**: 搭建 `quantrust-gateway`，实现用户注册与登录 (JWT)。
    - [ ] **前端**: 初始化 React + Vite + TailwindCSS 项目，实现登录/注册页面和主布局。
    - [ ] **数据库**: 完成 PostgreSQL 的 `users` 表设计与部署。

- **Sprint 3-4: 数据服务与行情展示 (2周)**
    - [ ] **后端**: 开发 `quantrust-data` 服务，实现从 AkShare/Tushare 获取日线/分钟线数据。
    - [ ] **后端**: 设计并部署 ClickHouse 的 `stock_candles` 表。
    - [ ] **后端**: 实现数据清洗和入库的定时任务。
    - [ ] **前端**: 开发行情看板页面，集成 TradingView 或 ECharts 实现 K 线图展示。
    - [ ] **前端**: 实现股票搜索和自选股列表功能。

- **Sprint 5-6: 策略 IDE 与回测引擎 (2周)**
    - [ ] **后端**: 开发 `quantrust-strategy` 服务，集成 PyO3，实现 Python 策略的沙箱执行环境。
    - [ ] **后端**: 开发 `quantrust-backtest` 服务，实现向量化回测引擎。
    - [ ] **前端**: 开发策略研发页面，集成 Monaco Editor 作为 Web IDE。
    - [ ] **前端**: 实现策略的创建、保存、回测参数配置和回测报告展示。

- **Sprint 7-8: 模拟交易与监控 (2周)**
    - [ ] **后端**: 开发 `quantrust-trade` 服务，实现模拟盘的订单撮合逻辑。
    - [ ] **后端**: 开发 `quantrust-risk` 服务，实现基础的风控检查（资金、持仓）。
    - [ ] **前端**: 开发实时监控页面，通过 WebSocket 展示模拟盘的持仓、盈亏和日志。
    - [ ] **运维**: 部署 Prometheus 和 Grafana，为核心服务添加基础监控。

### **第二阶段: 功能增强与体验优化 (预计 6 周)**

**目标**: 完善核心功能，优化用户体验，为实盘交易做准备。

- [ ] **回测增强**: 实现事件驱动回测引擎，提供更精细的回测能力。
- [ ] **回测增强**: 开发参数优化功能，支持网格搜索和结果可视化。
- [ ] **策略增强**: 实现策略版本管理和差异对比功能。
- [ ] **风控增强**: 实现更复杂的风控规则配置和告警系统。
- [ ] **UI/UX 优化**: 完善暗色/亮色主题切换，优化移动端适配。
- [ ] **社区功能**: (可选) 增加策略分享、社区讨论等功能。

### **第三阶段: 实盘交易与商业化 (预计 4 周)**

**目标**: 对接真实券商接口，上线实盘交易功能，并探索商业化模式。

- [ ] **实盘对接**: 实现主流券商（如华泰、中信）的交易接口适配。
- [ ] **实盘功能**: 开发安全的券商账户绑定、实盘部署流程。
- [ ] **安全审计**: 对整个系统进行全面的安全审计和渗透测试。
- [ ] **商业化**: 设计不同用户角色的订阅套餐（如 Leo 免费，Isabella/David 收费）。
- [ ] **运维强化**: 完善日志、监控和告警系统，确保 99.9% 的可用性。
